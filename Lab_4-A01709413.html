¿Qué diferencias y semejanzas hay entre Java y JavaScript?

Diferencias

  • Java es un lenguaje de programación OOP, mientras que Java Script es un lenguaje de scripts OOP.
  • Java crea aplicaciones que se ejecutan en una máquina o explorador virtual, mientras que el código JavaScript sólo se ejecuta en un explorador.
  • El código Java necesita compilación, mientras que el código JavaScript está en todo el texto.
  • Java es un lenguaje compilado2, JavaScript es un lenguaje interpretado.
  • Java es fuertemente tipado (no hace conversión de tipos implícitamente), JavaScript es débilmente tipado (hace conversión de tipos implícitamente)
¿Qué métodos tiene el objeto Date?

Alguno de los métodos son:

      Date.prototype.getDate(): Retorna el día del mes (1 a 31) para la fecha especificada acorde a la hora local.
      Date.prototype.getHours(): Retorna la hora para la fecha especificada acorde a la hora local.
      Date.prototype.getUTCDate(): Retorna día del mes para la fecha especificada acorde hora local
      Date.prototype.toISOString(): Convierte la fecha en una cadena de texto
      Date.prototype.toLocaleDateString():Retorna una cadena con una representación sensible a la localización de la fecha basada en la configuración del sistema.
¿Qué métodos tienen los arreglos?

Algunos métodos son:

      Array.prototype.at(): Recibe un valor numérico entero y devuelve el elemento en esa posición, permitiendo valores positivos y negativos. Los valores negativos contarán desde el último elemento del array.
      Array.prototype.concat(): Se usa para unir dos o más arrays.
      Array.prototype.copyWithin(): Transfiere una copia plana de una sección a otra dentro del mismo array ( o contexto similar ), sin modificar su propiedad length y lo devuelve.
      Array.prototype.find(): Sirve para encontrar un termino dentro de un arreglo.
      Array.from(): Crea una nueva instancia de Array a partir de un objeto iterable.

¿Qué implicaciones tiene utilizar variables globales dentro de funciones?

Esras variables globales son visibles y disponibles para todas las sentencias de un script. Esto no siempre es recomendable debido a que se pueden equivocar y cambiarles el valor sin que se den cuenta, causando esto fallas o información erronea en tu Web.
