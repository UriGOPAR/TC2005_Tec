PREGUNTAS DE HTML:

¿Cuál es la diferencia entre Internet y la World Wide Web?

La World Wide Web (WWW) es una red informática de alcance mundial, el cual contiene sistemas de hipertextos e hipermedios (como páginas web) interconectados entre sí, a los cuales se puede acceder con Internet y software especializado, mientras que el Internet es solamente una red de computadoras muy grande alrededor del mundo

¿Cuales son las partes de la URL?

Las partes de la URL son 3:
1.-Protocolo y subdominio: https://www
2.-Dominio: Este esta compuesto por dos elementos (Nombre de dominio y la extensión de dominio)
3.-Ruta Completa: Esta extensión contiene etiquetas y parámetros que sirven para precisar el contenido exacto que va a mostrar el navegador. Esta dividida por "/"

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET: Solicita una representación de un recurso específico. Solo las peticiones recuperan datos
HEAD: Pide una respuesta idéntica a la de la petición GET, pero sin el cuerpo
POST: Se utiliza para enviar una entidad a un recurso específico, causando un cambio en el servidor
PUT: Reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición
PATCH: Sirve para aplicar modificaciones parciales a un recurso
DELETE: Sirve para borrar recursos en específico

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Para la creación de un formulario se utiliza el método (form), y los dos atributos más importantes que es cuando el usuario envia el formulario es ACCIÓN y MÉTODO:

  1. Acción: Es el que decide el lugar donde se enviaran los datos. Su valor debe ser una URL válida
  2. Método: Define como se enviaran los datos; pero los más conocidos es por el metodo GET y POST

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

  • HTML5 tiene muchas etiquetas nuevas que no estaban presentes en html4 como lienzo, video, audio
  • HTML5 Tiene un mejor manejo de errores.
  • En Html5, se eliminó la etiqueta de applet que se usó para mostrar los applets en un navegador web para hacer esto en la html5 se introdujo una nueva etiqueta, en html4 solo estaba presente la etiqueta de applet para mostrar los applets en los navegadores web.
  • En html5 para soporte de comunicación de cliente y servidor, contiene sockets web que permiten la comunicación dúplex completa entre clientes y servidores, mientras que en caso de comunicación html4 entre el cliente y el servidor se realizó a través de transmisión y sondeo largo, ya que no había sockets web disponibles en HTML4.

SOBRE EL CICLO DE VIDA Y DESARROLLO DE LOS SISTEMAS DE INFORMACIÓN

¿Cuál es el ciclo de vida de los sistemas de información?


Los ciclos de vida de los sistemas son:

  1. Planificación: Investigación en un principio
  2. Análisis de los Requerimientos
  3. Diseño: Estructura general que tendra el sistema
  4. Implementación: Seleccionar las herramientas adecuedas para facilitar el cumplimiento del trabajo
  5. Pruebas: La detección de ERRORES
  6. Instalación: Planificar el entorno en el que el sistema debe funcionar
  7. Uso y Mantenimiento: Eliminar defectos, Adaptar el programa a nuevas necesidades y Añadirle nueva funcionalidades
  8. Delimitación del ámbito del proyecto
  9. Estudio de Viabilidad
  10. Analisis de Riesgo

¿Cuál es el ciclo de desarrollo de sistemas de información?

Los ciclos de desarrolo son:

      Fase de planificación
      Fase de Análisis
      Fase de desarrollo
      Fase de Periodo de Pruebas
      Fase implementación
      Mantenimiento
